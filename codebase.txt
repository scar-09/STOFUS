3. KEY FLOWS
────────────────────────────────────────────────────────────────────────

 1. Navigation → webNavigation.onCommitted → checkAndBlock()
    → domain match → redirect to blocked.html?site=X&type=Y&url=Z

 2. Keyword match in content.js → redirect to kw-blocked.html?kw=X&url=Y

 3. Popup "Block site" → chrome.storage.local.set({ blockedSites })
    → background catches via storage.onChanged → creates focusSession

 4. Friction unlock → removes from blockedSites, reloads original tab,
    closes friction tab. No temp unlock created.

 5. Cleanup alarm (1 min) → removes expired timer blocks,
    updates stats (focusTime, streaks, dailyFocusCompleted)

 6. Keyword bypass → kw-blocked.js stores keywordBypass[tabId],
    creates alarm for expiry → cooldown → cleanup

 7. Format toggle → rebuildTimePickers() in popup.js:
    12h ↔ 24h. Hidden inputs are always 24h (source of truth).

 8. Block page states → blocked.html has two-state layout:
    blockedState (timer countdown) ↔ unblockedState (protection complete)

 9. Duration unit → click-to-cycle button: Minutes → Hours → Days → Minutes

10. Time pickers → editable text inputs with input validation and
    blur-based clamping. No dropdowns.

11. Content.js tracks keyword attempts in dailyAttempts[keywords]
    before redirecting to kw-blocked.html.

12. Popup attempts section split into "Websites" and "Keywords"
    subsections, each showing attempt counts per item.

13. Friction page has 3 phases:
    a) Reflection (30s) — "What were you planning to do on this site?"
    b) Friction timer (60s/180s/300s) — timer blocks only
    c) Phrase input — "I choose to unlock this website"

14. Schedule blocks skip the friction timer, showing end time info instead.

15. Keyword bypass cleared on keyword removal from popup.

9. DESIGN DECISIONS
────────────────────────────────────────────────────────────────────────

  - All times stored internally in 24h format. Hidden inputs (startTimeInput,
    endTimeInput) are the single source of truth. Time picker inputs and
    meridiem buttons are UI-only, updated via rebuildTimePickers() on format
    toggle.

  - Timer blocks: strict, no bypass possible. Schedule blocks: no temp
    unlock either. Friction page is the ONLY removal path.

  - Keyword bypass: 3 min active, 45 min cooldown. Per-tab tracking.
    Bypass cleared when keyword is removed from popup.

  - Focus session: created when blockedSites become active, destroyed when
    all blocks expire/are removed. Duration = max block duration or schedule
    window size.

  - Undo: 5-second window, reverts last block addition including stats and
    dailyAttempts.

  - Accordion: single-open behavior via checkbox hack.

  - Interruption pages (blocked/kw-blocked/friction) are not dashboards.
    They are checkpoints: inform, enforce, guide. No gamification, no
    celebration, no emotional language.

  - Friction page has 3 sequential phases:
    1. Reflection (30s) — "What were you planning to do on this site?"
    2. Friction timer (60s/180s/300s) — timer blocks only
    3. Phrase input — "I choose to unlock this website"
    Schedule blocks: reflection → schedule end time info → phrase input.

  - Monochrome command-console palette:
    Background:   #050505
    Surface:      #0F0F0F
    Elevated:     #151515
    Borders:      #2A2A2A
    Primary text: #F5F5F5
    Secondary:    #B3B3B3
    Muted:        #7A7A7A
    Active:       #D4D4D4
    Semantic:     #22C55E (success), #F59E0B (warning), #EF4444 (danger)
    Font:         JetBrains Mono
    Radius:       4px
    Effects:      None — no glow, no blur, no glass, no neon, no particles

  - Time pickers: editable text inputs (no dropdowns). Non-digit chars
    stripped on input; range clamped on blur (hour: 0-23 or 1-12, minute:
    0-59). Empty fields default to 12 (12h) or 00 (24h).

  - Duration unit: click-to-cycle button (Minutes → Hours → Days → Minutes).
    Timer value resets to unit default on each cycle.

  - Popup status bar: monochrome text (white for success, amber for undo,
    no cyan). Fades out smoothly instead of instant clear.

  - Stats consolidated into two objects: stats (aggregate/lifetime values)
    and dailyAttempts (per-site + per-keyword blocked navigation counts).
    Nested as { sites: {...}, keywords: {...} } under each date key.
    Old flat format auto-migrated on first popup render.

  - All transition: all replaced with explicit property lists.
    Accordion content uses consistent 0.3s ease with fixed padding
    (no padding transition to prevent content shift).

  - will-change hints added on animated elements (accordion content,
    patterns body, collapsible lists, protection dot) for GPU-accelerated
    transitions.

  - Read-only status text has user-select: none to prevent accidental
    text selection. Site URLs and Remove buttons remain selectable.

  - Welcome page is a terminal briefing, not a marketing page. First-run
    document with bullet-point sections. No card, no glass, no colors.
